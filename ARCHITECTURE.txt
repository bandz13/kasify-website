================================================================================
                    KASIFY: CLIENT/SERVER ARCHITECTURE
================================================================================

================================================================================
OVERVIEW
================================================================================

Kasify is a FULL-STACK web application with clear separation between:
- FRONTEND (Client-Side) - What users see and interact with
- BACKEND (Server-Side) - Where data lives and is managed

================================================================================
FRONTEND (CLIENT-SIDE) - What runs in the browser
================================================================================

Files & Technologies:
- index.html         → Main homepage (Tailwind CSS + JavaScript)
- sellers.html       → Seller directory page with search/filters
- seller-profile.html → Individual seller profile pages
- JavaScript (ES6+) → Dynamic interactions, API calls

Responsibilities:
✓ Display user interface (UI)
✓ Handle user interactions (clicks, form submissions)
✓ Fetch data from Firebase
✓ Render dynamic content from Firebase
✓ Local form validation before sending to server
✓ Search and filter functionality (client-side)

Example Flow:
1. User clicks "View All Sellers" button
2. Browser loads sellers.html
3. JavaScript fetches all sellers from Firebase using getDocs()
4. Results display in grid with search/filter capability
5. User clicks a seller card
6. JavaScript navigates to seller-profile.html?id=mpho-web-studio

================================================================================
BACKEND (SERVER-SIDE) - Firebase Firestore
================================================================================

What is Firebase?
- Cloud database that stores seller data, products, reviews, deals
- Real-time data synchronization
- Built-in security rules
- No traditional server code needed (Serverless)

COMMUNICATION FLOW: How Client & Server Interact
================================================================================

SCENARIO 1: User Views All Sellers

1. [BROWSER] User clicks "View All Sellers" button
2. [BROWSER] sellers.html loads
3. [BROWSER] JavaScript runs loadAllSellers() function

5. [FIREBASE] Returns all sellers from cloud database
6. [BROWSER] JavaScript renderSellers(allSellers)
7. [BROWSER] HTML grid displays all sellers with cards
8. [USER] Sees beautiful grid of sellers

================================================================================
DATA FLOW EXAMPLE: Complete Journey
================================================================================

STEP 1: Seller Registers Their Business
- Admin adds seller to Firebase SELLERS collection
- Data saved: businessName, category, location, logo, etc.

STEP 2: Homepage Loads
- Browser downloads index.html (static teasers)
- Shows 3 sample sellers
- "View All Sellers" button ready

STEP 3: User Clicks "View All Sellers"
- Browser loads sellers.html
- JavaScript: getDocs(collection(db, 'sellers'))
- Firebase returns ALL seller documents
- JavaScript maps each seller to a clickable card
- User sees grid of all sellers

STEP 4: User Clicks on "Mpho Web Studio"
- onclick="window.location.href='seller-profile.html?id=mpho-web-studio'"
- Browser navigates with seller ID in URL
- seller-profile.html loads

STEP 5: seller-profile.html Runs JavaScript
- Extracts sellerId from URL: "mpho-web-studio"
- Makes 4 parallel Firebase queries:
  a) getDoc(db, 'sellers', 'mpho-web-studio')
  b) getDocs(where('sellerId', 'mpho-web-studio')) from PRODUCTS
  c) getDocs(where('sellerId', 'mpho-web-studio')) from REVIEWS
  d) getDocs(where('sellerId', 'mpho-web-studio')) from DEALS

STEP 6: Firebase Returns Data
- Seller: {businessName, category, about, hours, ...}
- Products: [{name, price, description, ...}]
- Reviews: [{rating, comment, reviewer, ...}]
- Deals: [{title, discount, ...}]

STEP 7: JavaScript Renders Profile Page
- renderSellerProfile(seller) → Shows header, cover, about
- renderProducts(products) → Shows product grid
- renderReviews(reviews) → Shows ratings
- renderDeals(deals) → Shows promotions

STEP 8: User Sees Immersive Seller Website
- Professional design
- All seller's data displayed beautifully
- WhatsApp buttons to contact
- Can see products, reviews, deals

STEP 9: User Clicks "Order on WhatsApp"
- onclick opens WhatsApp with pre-filled message
- Direct conversation with seller starts

================================================================================
SUMMARY: CLIENT vs SERVER
================================================================================

CLIENT-SIDE (Browser - What You Code in index.html, sellers.html, etc):
├─ HTML structure
├─ Tailwind CSS styling
├─ JavaScript event listeners (clicks, forms)
├─ Firebase SDK calls (getDocs, getDoc, query, where)
├─ Data rendering and DOM manipulation
└─ User interaction logic

SERVER-SIDE (Firebase Cloud - Where Data Lives):
├─ Firestore Database collections
├─ Data persistence (data stored permanently)
├─ Security rules (who can read/write)
├─ Data retrieval APIs (queries)
└─ Real-time synchronization

WORKFLOW:
Client makes request → Firebase processes → Returns data → Client renders

This is a COMPLETE WEB STACK:
✓ Frontend: HTML, CSS, JavaScript (in browser)
✓ Backend: Firebase Firestore (in cloud)
✓ Communication: HTTPS/REST API (Firebase SDK)
✓ Database: Cloud Firestore (persistent storage)
